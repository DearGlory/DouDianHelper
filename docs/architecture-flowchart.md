## 一、主流程

```
┌─────────────────────────────────────────────────────────────┐
│                        程序启动                              │
│  main.py → parse_args() → load_config()                     │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  读取 Excel 订单 → 检查暂停状态文件                          │
│  ┌─ 有暂停状态 + 无显式 limit/workers → 恢复任务            │
│  ├─ 有暂停状态 + 有显式 limit/workers → 忽略并清理旧状态     │
│  └─ 无暂停状态 → 新任务启动                                  │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  启动后台任务                                                │
│  ├─ _watch_for_exit_command()  (q/quit/exit 退出监听)       │
│  └─ _resource_monitor()       (资源使用监控)                 │
└──────────────────────────┬──────────────────────────────────┘
                           ▼
            ┌──────────────────────────┐
            │  while order_ids 主循环   │◄─────────────────────┐
            └────────────┬─────────────┘                       │
                         ▼                                     │
        ┌────────────────────────────────┐                     │
        │  登录预热                       │                     │
        │  ensure_cdp_storage_state()    │                     │
        │  (详见"登录态流程")             │                     │
        └────────────┬───────────────────┘                     │
                     ▼                                         │
        ┌────────────────────────────────┐                     │
        │  按 restart_every 拆分 chunks   │                     │
        └────────────┬───────────────────┘                     │
                     ▼                                         │
        ┌────────────────────────────────┐                     │
        │  for chunk in chunks:          │                     │
        │  ┌─ 创建 BrowserWorker ×N     │                     │

        │  ├─ 填充订单队列               │                     │
        │  └─ 并行处理订单               │                     │
        └────────────┬───────────────────┘                     │

        │  ④ 发送消息                    │                     │
        │  ⑤ ESC 退出会话               │                     │

                     ▼                                         │

        │                                        │             │

        │  会话中断?─────────→ 重连浏览器 ────────┤             │
        │  正常完成?─────────→ 下一个 chunk       │             │
```

## 二、风控流程

```
        ┌────────────────────────────────────────────┐
        │  _raise_if_risk_control_detected(page)     │
        │  每次关键操作前/后都会调用                    │
        └──────────────┬─────────────────────────────┘
                       ▼
        ┌──────────────────────────────┐
        │  检测硬风控 (验证码/人机)     │
        │  _detect_risk_control()      │
        └──────┬───────────────────────┘
               │ 命中?
               ├─ 是 → 立即抛出 RISK_CONTROL_DETECTED
               │       (不重试，直接进入暂停)
               ▼
        ┌──────────────────────────────┐
        │  检测软风控 (请刷新/限流)     │
        │  _detect_soft_risk_control() │
        └──────┬───────────────────────┘
               │ 命中?
               ├─ 否 → 正常继续
               ▼
        ┌──────────────────────────────────────┐
        │  详情包含"请刷新"?                    │
        │  ├─ 否 → 立即抛出 SOFT_RISK_CONTROL  │
        │  ▼                                   │
        │  是 → 进入 3 次重试循环               │
        │  ┌─ 等待 3 秒                        │
        │  ├─ 重新检测软风控                    │
        │  ├─ 消失? → 继续执行 ✓               │
        │  └─ 3 次都在? → 抛出 SOFT_RISK_CONTROL│
        └──────────────────┬───────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  worker 捕获风控异常                  │
        │  → risk_control_event.set()          │
        │  → 当前 chunk 提前结束               │
        └──────────────────┬───────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  主流程检测到 risk_detected           │
        │  ① 保存运行时状态 (pause_state.json) │
        │  ② 释放 Edge 资源 (kill_edge)        │
        │  ③ 进入 _pause_before_resume()       │
        │     等待 risk_control_pause_seconds   │
        │     (默认 1800s = 30 分钟)            │
        └──────────────────┬───────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  暂停结束                             │
        │  ① 读取 pause_state.json 恢复状态    │
        │  ② 重新读取 Excel 订单               │
        │  ③ 回到主循环顶部                     │
        │     → 重新走登录预热                  │
        │     → 继续剩余订单                    │
        └──────────────────────────────────────┘
```

## 三、登录态获取/失效恢复流程

```
        ┌────────────────────────────────────────────┐
        │  ensure_cdp_storage_state(force_refresh)   │
        │  入口：每轮主循环开始时调用                   │
        └──────────────────┬─────────────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  判断模式                             │
        │  ├─ launch + dedicated + 非force     │
        │  │  → return None (worker自行管理)   │
        │  ├─ use_real_user_profile=true        │
        │  │  → 进入真实 Profile 分支           │
        │  └─ 其他 (cdp / force_refresh)       │
        │     → 进入通用分支                    │
        └──────┬───────────────┬───────────────┘
               ▼               ▼
    ┌─────────────────┐  ┌─────────────────────────┐
    │ 真实 Profile 分支│  │ 通用分支                 │
    └────────┬────────┘  └────────────┬────────────┘
             ▼                        ▼
    ┌─────────────────┐  ┌─────────────────────────┐
    │ CDP 已就绪?      │  │ headless + 有bootstrap? │
    │ ├─ 否:           │  │ ├─ 是: 验证有效性       │
    │ │  强杀旧 Edge   │  │ │  ├─ 有效 → 复用 ✓    │
    │ │  重启 CDP 9222 │  │ │  └─ 失效 → 继续 ↓    │
    │ │  等待就绪      │  │ └─ 否: 继续 ↓           │
    │ └─ 是: 继续 ↓    │  └────────────┬────────────┘
    └────────┬────────┘               ▼
             ▼               ┌─────────────────────────┐
    ┌─────────────────┐      │  有头人工登录预热         │
    │ force_refresh?   │      │  ① 强杀旧 Edge          │
    │ ├─ 否: 尝试复用  │      │  ② 启动有头 Edge + 9222 │
    │ │  bootstrap有效?│      │  ③ 打开飞鸽页面          │
    │ │  ├─ 是→复用 ✓  │      │  ④ 等待用户登录 + Enter  │
    │ │  └─ 否→继续 ↓  │      │     (或自动检测登录)     │
    │ └─ 是: 继续 ↓    │      │  ⑤ 保存 bootstrap state │
    └────────┬────────┘      │  ⑥ 如 headless:          │
             ▼               │     杀有头→重启 headless  │
    ┌─────────────────┐      └─────────────────────────┘
    │ 从 CDP 会话提取  │
    │ ├─ 成功→保存 ✓   │
    │ └─ 失败→转入     │
    │   有头人工登录 ──────────────────→ (同右侧)
    └─────────────────┘
```

## 四、登录态失效自动恢复（运行中）

```
        ┌────────────────────────────────────────────┐
        │  Worker 启动 / 运行中                       │
        │  _ensure_logged_in(page) 检测到未登录       │
        │  → 抛出 LOGIN_STATE_MISSING                 │
        └──────────────────┬─────────────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  _process_chunk 或 _order_worker     │
        │  捕获 LOGIN_STATE_MISSING            │
        │  → session_broken_event.set()        │
        │  → login_refresh_required = True     │
        └──────────────────┬───────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  主流程检测到 login_refresh_required  │
        │  → force_refresh_login = True        │
        │  → order_ids = 剩余订单              │
        │  → break 当前 chunk 循环             │
        └──────────────────┬───────────────────┘
                           ▼
        ┌──────────────────────────────────────┐
        │  回到主循环顶部                       │
        │  ensure_cdp_storage_state(           │
        │      force_refresh=True              │
        │  )                                   │
        │  → 强制进入有头人工登录预热           │
        │  → 用户重新登录                       │
        │  → 保存新 bootstrap state            │
        │  → 继续剩余订单                       │
        └──────────────────────────────────────┘
```

## 五、浏览器启动模式对比

| 维度 | `launch` 模式 | `cdp` 模式 |
|------|--------------|-----------|
| 含义 | Playwright 自己启动浏览器 | 连接已有的 CDP 调试浏览器 |
| 浏览器生命周期 | 程序完全控制 | 外部启动，程序只接管 |
| 登录态来源 | storage_state 文件注入 | 复用已有浏览器会话 |
| 真实 Profile | 支持但需 CDP 预热 | 天然支持 |
| headless | 直接支持 | 需要 `--headless=new` 参数 |
| 适用场景 | 正式批量跑单 | 登录预热 / 真实环境复用 |
| 当前项目中 | worker 正式执行层 | 登录态准备层 |

### 实际运行时的模式切换
```
启动 → 登录预热 (CDP/有头) → 保存登录态 → 正式执行 (launch/headless)
                                              ↑
风控恢复 / 登录失效 → 重新预热 (CDP/有头) ────┘
```

## 六、关键文件职责

| 文件 | 职责 |
|------|------|
| `main.py` | 主流程编排：订单调度、chunk 轮转、风控暂停恢复、登录刷新触发 |
| `browser_worker.py` | 单订单处理：飞鸽搜索、资格校验、消息发送、风控检测 |
| `capture_storage_state.py` | 登录态管理：预热、提取、验证、有头登录兜底 |
| `launch_edge.py` | Edge 进程管理：启动、杀进程、CDP 等待、Profile 解析 |
| `excel_reader.py` | Excel 读写：读取订单、删除已处理行、导出复核文件 |
| `pause_state.py` | 暂停状态持久化：保存/读取/清理 `*.pause_state.json` |
| `logger_utils.py` | 日志配置 |
| `config.json` | 运行时配置（不入 git） |
| `start_edge.bat` | 手动启动 Edge 的快捷脚本 |
